---
name: 整理需求/代码/笔记时分类保存到memory
description: 当用户说"整理需求"或让整理代码/笔记时，将整理结果按类型分类保存到memory，以便未来"通用设计"请求时按类型参考和生成模板
type: feedback
---

当用户说"整理需求"或让整理某个代码、整理笔记时，将整理的内容按类型分类保存到 memory（user 或 project 目录）。

**Why:** 用户希望积累分类化的知识素材，未来说"需要通用设计"时，可以按类型检索已保存的素材，参考历史整理结果来设计和生成模板，而非每次从零开始。

**How to apply:**
- 触发词："整理需求"、"整理代码"、"整理笔记"、"整理xxx"
- 整理完成后，判断内容类型（如：需求模板、接口设计、数据模型、业务流程、编码规范、架构模式、测试规范等）
- 按类型分类保存到对应 memory 目录：
  - 跨项目通用的 → user 目录（`C:\Users\admin\.qwen\memories\`）
  - 项目专属的 → project 目录（`C:\Users\admin\.qwen\projects\<project>\memory\`）
- 保存时在 MEMORY.md 索引中添加一行指针，标注类型关键词
- 未来用户说"通用设计"/"需要通用设计"时，先检索 memory 中已保存的同类型素材，作为参考基础来生成模板
- 与已有的"设计模板生成时加入业务理解"配合：先检索历史素材，再结合业务场景补充
- **整理结束后的提示**：每次整理并保存完成后，在回复结尾明确提示用户："已整理完成并保存到memory，后续'通用设计'时可按类型参考"，让用户知道素材已积累可用
