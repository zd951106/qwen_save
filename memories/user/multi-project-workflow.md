The user works on multiple projects simultaneously, each in its own IDEA window, and expects knowledge from one project's conversation to carry over to others.

**How to apply:** When the user shares a fact that is not specific to the current project (personal preferences, general workflow conventions, role/background), save it as USER memory (cross-project) rather than PROJECT memory, so it is visible in the other IDEA projects too. Reserve PROJECT memory for facts tied to this repo only. When unsure which scope the user wants, ask — they are aware of the two-tier system.
